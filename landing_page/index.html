Why Sign Your Agents

AI agents wield real tools.
Their integrity matters.

When an agent can search the web, execute code, or access sensitive data, any undetected change to its configuration is a security risk. Signing creates a verifiable chain of trust.

🔒

Tamper Detection

Catch unauthorized changes instantly. If anyone modifies a tool's name, description, or parameters — the signature breaks and you know immediately.

👤

Attribution & Identity

Prove exactly who approved an agent configuration. Ed25519 signing and JWT identity tokens create an auditable trail from developer to deployment.

🚀

Deploy with Confidence

Gate deployments on valid signatures. Before any agent goes to production, verify that its tools and configuration match what was approved in development.

🔧

Framework-Agnostic

Works with LangChain, LangGraph, CrewAI, or plain dictionaries. Auto-discovers metadata from any supported framework — no lock-in.

📋

Compliance & Audit

Build audit trails with timestamped signature files and a central registry server. Meet compliance requirements for AI governance and operational transparency.

⚖

Order-Independent

Reorder tools or agents in your code freely — the signature stays valid. Only semantic changes to the configuration break the signature.

How To Sign

Three steps to a
signed agent.

Install the package, register your tools, and sign. It takes less code than your import statements.

01

Install & Initialize

Install from PyPI and create a signer with your preferred signing mode — HMAC, Ed25519 keypair, or JWT identity.

# Install pip install agent-signing # Initialize with a keypair from agent_signing import ( AgentSigner, generate_keypair ) private_key, public_key = generate_keypair() signer = AgentSigner( private_key=private_key )
02

Register Tools & Agents

Add any combination of tools and agents. Pass framework objects directly or use plain dictionaries — metadata is extracted automatically.

# Add framework tools directly signer.add_tool(search_tool) signer.add_tool(calculator_tool) # Or add agents (auto-discovers tools) signer.add_agent(react_agent) # Or use plain dicts signer.add_tool({ "name": "search", "description": "Search the web", "parameters": {"query": "str"} })
03

Sign & Publish

Generate a signature, write it to a file, and optionally publish to a shared registry for team-wide verification.

# Sign to a file signer.sign_to_file( "agent_signature.json" ) # Publish to a shared registry signer.publish( "http://registry:8000", path="agent_signature.json" )
Verification

Verify before
you deploy.

Rebuild the same agent configuration on any machine and verify it against the original signature. If anything changed — a renamed tool, a modified parameter, a swapped model — verification fails.

  • ✓ Verify from a signature string in memory or a JSON file on disk
  • ✓ Only the public key is needed to verify — private keys never leave the signer
  • ✓ Decoded JWT identity claims are returned on successful verification
  • ✓ Query the registry server by hash to inspect any published signature
verify.py
from agent_signing import AgentSigner # Rebuild the same setup on any machine verifier = AgentSigner(public_key=public_key) verifier.add_tool(search_tool) verifier.add_tool(calculator_tool) # Verify against the signature file result = verifier.verify_file( "agent_signature.json" ) if result.valid: print("✓ Configuration verified") print(f" Signed by: {result.identity}") else: print(f"✗ {result.reason}") # Or look up a published signature # GET /signatures/{hash}
Signing Modes

Choose your trust model.

Three signing modes for different security requirements. Combine Ed25519 with JWT for both cryptographic proof and identity context.

Shared Secret

HMAC

The simplest mode. Both signer and verifier share a secret key. Best for single-team setups where key distribution is easy.

AgentSigner(secret="my-key")
Asymmetric Keypair

Ed25519

Sign with a private key, verify with the public key. Proves who signed the configuration. Private keys never leave the signing machine.

AgentSigner(private_key=key)
Identity Token

JWT

Attach an OAuth/OIDC identity token. On verification, decoded claims like subject and issuer are returned. Combine with Ed25519 for full provenance.

AgentSigner(identity_token="eyJ...")
